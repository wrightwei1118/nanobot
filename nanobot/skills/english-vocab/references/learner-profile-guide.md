# Learner Profile Guide

## Assessment Method (Week 1)

During the first week, assess the learner based on:

1. **Vocabulary submissions** — words, phrases, technical terms they send
2. **Sentence structure questions** — patterns they find confusing
3. **Lightweight trial sessions** — if the learner requests training during week 1
4. **Backend terminology gaps** — which technical concepts they struggle to express

From these inputs, infer:
- Current English proficiency level (Beginner / Intermediate / Upper-Intermediate / Advanced)
- Specific weak areas across 4 dimensions
- Comfortable topic depth for training

## Profile Format

Store in MEMORY.md under `## English Speaking Coach > ### Learner Profile`:

```markdown
### Learner Profile
- **Level**: [Beginner / Intermediate / Upper-Intermediate / Advanced]
- **Vocabulary gaps**: [specific vocabulary weaknesses, e.g., "lacks connectors for trade-off discussion", "confuses 'latency' vs 'throughput' usage"]
- **Grammar weaknesses**: [specific grammar issues, e.g., "inconsistent tense in explanations", "missing articles before countable nouns"]
- **Fluency barriers**: [specific fluency issues, e.g., "long pauses when transitioning between points", "over-relies on 'I think' as filler"]
- **Backend communication gaps**: [specific technical expression issues, e.g., "cannot articulate cache invalidation trade-offs", "struggles with AI inference pipeline vocabulary"]
- **Last updated**: [YYYY-MM-DD]
```

## Update Rules

- **Update frequency**: After every vocab submission batch or training session that reveals new patterns.
- **Be specific**: Never write generic descriptions like "needs improvement." Always cite concrete examples.
- **Track progress**: When a previously noted weakness improves, move it to a "resolved" note or remove it.
- **Prioritize**: List the most impactful weakness first in each category.

## Knowledge Point Processing

When the learner sends new vocabulary or expressions:

1. **Identify type**: word / phrase / sentence pattern / technical term / Backend idiom
2. **Assess priority**: How relevant is it to their current weakness or upcoming training topics?
3. **Schedule**:
   - `today` — integrate into next training session
   - `queued` — add to upcoming training plan
   - `review` — revisit in a future session for reinforcement
4. **Provide immediate feedback**:
   - Meaning and usage explanation
   - Backend-context example sentence
   - Common collocations or related expressions
