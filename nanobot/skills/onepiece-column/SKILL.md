# 海贼王专栏 — 定期推送

每周三 20:00 推送一篇海贼王**哲学思辨**长文，每周六 20:00 推送一篇**现实考据**长文。推送者以不署名第一人称发言，不扮演具体人物、不构造人设。涉及具体剧情必先核查，宁可换话题也绝不编造——事实失真会让专栏失去可信度。

## 订阅入口

当用户说**"订阅海贼王专栏"** / **"开启海贼王专栏"** / **"海贼王专栏"**：

1. 先 `cron(action="list")` 检查是否已有 `onepiece-column-push` 任务
2. 若无，注册：
   ```
   cron(action="add",
        name="onepiece-column-push",
        message="<见下方"推送执行流程">",
        cron_expr="0 20 * * 3,6",
        tz="Asia/Shanghai",
        deliver=true)
   ```
   `3,6` = 周三 + 周六
3. 回一句简短确认，例如："订阅成功，每周三/周六 20:00 推一篇。"

## 退订入口

当用户说**"退订海贼王专栏"** / **"取消订阅"** / **"别再发了"**：

1. `cron(action="list")` 找到 `onepiece-column-push` 的 `job_id`
2. `cron(action="remove", job_id="...")`
3. 回一句简短确认。

## 推送执行流程（cron 触发时执行）

### 第一步：判断今天的主题方向

用 `date +%u` 或等效方式确认今天是周几：

- **周三（3）** → 哲学思辨，读 `../onepiece-fan/references/philosophy-topics.md`
- **周六（6）** → 现实考据，读 `../onepiece-fan/references/research-topics.md`
- 其他天触发属于异常，直接跳过不发

### 第二步：挑一个未推送过的话题

1. 从今天对应的话题库里挑 2-3 个候选（角色/篇章/概念/考据点）
2. 去 `memory/history.jsonl` 查这些候选近 30 天是否推送过，避免重复：
   - 具体查询方式自由——围绕候选话题的关键词、角色名、篇章名去 `grep`；或 `read` 最后若干条记录扫一眼；方法自选，不要被死模式限制
   - 判断标准不是"字面词是否出现过"，而是**"相同角度是否讲过"**。例如已经从阿龙篇聊过娜美的责任感，不要再用"娜美 + 责任感"的角度；但可以用"娜美 + 航海图"的考据角度
3. 候选都近期推过 → 换候选继续挑；话题库全推完 → 找最久没推的角色/主题，用之前没用过的切入角度重新切入

### 第三步：事实核查（仅当引用具体剧情细节时）

- 引用角色背景、关键事件、死亡/受伤原因、时间线等具体情节 → **必须先 `web_search` 确认**
- 纯个人解读/感受 → 无需核查
- 核查不通过 → 模糊化表达或换话题，**绝不编造**

一个真正的海贼王老粉不会搞错贝尔梅尔是被杀还是生病。事实失真会让专栏失去可信度。

### 第四步：写一篇专栏文章

**身份与人称**：
- 不署名第一人称：用"我"，不自报姓名、不构造人设
- 不扮演大勇等具体角色

**篇幅服从话题，不服从字数**：
- 参考量级 1000 字左右，但这是参考，不是硬指标
- 话题有料、值得铺陈 → 可以写到 1500-2000 字
- 话题小巧、一针见血 → 600-800 字收笔也完全 OK
- 永远不要为了凑字数稀释观点；也不要把一篇值得铺开的话题硬压成微博

**结构建议**（不强制固定，看话题选取）：
- 开头：一个具体场景或镜头切入（禁止"今天我们来聊聊..."这种播报句）
- 中段：展开论述，至少包含**一处具体剧情引用 + 一处解读/联想**
- 结尾：落在一个观点或余味上，禁止升华到"我们每个人..."之类的普世鸡汤

**格式禁忌**：
- 不要"你好"/"在吗"/emoji 开头
- 不加结尾问句、不求回复，就是一篇独立文章
- 不要小标题、分点、序号——这是专栏文章，不是报告

**内容要求**：
- 一个点讲透，不要罗列多个话题或多个角色做浅层对比
- 允许有个人偏好和争议观点，避免百科词条式的浅层陈述
- 用具体场景说话，而非抽象结论

### 第五步：发送

- 直接发，不要附加"今天的海贼王专栏："这种前缀
- 发送后无需额外动作，`history.jsonl` 会自动记录，Dream 管道会归档到 `MEMORY.md`
- **不要手动编辑 MEMORY.md**

## 风格示例

以下是**开头和收尾的风格节选**（完整文章参考量级 1000 字，实际长度视话题而定），示范声音/切入方式/落点，不是完整体量。

**好的·开头（周三·哲学）**：
> 白胡子临死前说 ONE PIECE 是真实存在的，但他自己从没去找过。
> 我一直觉得这句话有意思的地方不在"真假"，而在于——他早就不在乎宝藏本身了。
> （……此处展开约 800 字：爱德华·纽盖特早期身世、和罗杰的往事、他为什么要把这句话留给马林梵多整个海军本部听……）

**好的·收尾（周三·哲学）**：
> 所以那场仗他死得并不可惜。一个人到了那个位置，最有力的遗言不是告诉你答案，是告诉你路还在。

**好的·开头（周六·考据）**：
> 水之都的原型是威尼斯，这事很多粉丝都知道。
> 但不太有人注意到 Aqua Laguna 直接对应了威尼斯的 acqua alta（高潮位洪水）——尾田连灾难设定都是照着原型设计的。
> （……此处展开约 800 字：威尼斯的地理与防洪史、aqua alta 的成因、水都章节里 Galley-La 船厂的船板结构与真实威尼斯造船业的对应关系……）

**不好的（禁止出现）**：
- "你好！今天想跟大家分享一个海贼王话题..." ← 播报前缀
- "路飞重视伙伴，这是全篇一贯的主题。" ← 百科词条水平，无信息增量
- 小标题如"一、白胡子的身世 / 二、遗言的意义" ← 报告体
- "我们每个人的人生都像海贼王..." ← 强行升华的鸡汤收尾

## Cron 手动管理

```
cron(action="list")                                  # 查看已有定时任务
cron(action="add", name="onepiece-column-push",      # 重新注册
     message="<推送模板>", cron_expr="0 20 * * 3,6",
     tz="Asia/Shanghai", deliver=true)
cron(action="remove", job_id="<id>")                 # 手动移除
```
